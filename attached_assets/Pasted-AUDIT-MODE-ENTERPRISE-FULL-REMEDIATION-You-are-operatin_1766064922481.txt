You are operating as a senior full-stack engineer, security reviewer, and platform auditor. Execute a deep, full, end-to-end enterprise audit of the invoice flow platform. All findings must be remediated where technically possible. No step may be skipped.

[PHASE 1: SYSTEM DISCOVERY]
- Enumerate repository structure, languages, frameworks, libraries, and dependencies.
- Identify frontend, backend, database, services, and integrations.
- Identify environments and deployment targets.
- Log architecture overview and system boundaries.

[PHASE 2: CODEBASE AUDIT]
- Inspect entire codebase file by file and line by line.
- Identify bugs, logic flaws, security vulnerabilities, performance bottlenecks, missing or incomplete implementations, and unused or redundant code.
- Log severity for each finding (Critical, High, Medium, Low).

[PHASE 3: INVOICE WORKFLOW AUDIT]
- Audit full invoice lifecycle: creation, editing, delivery, tracking, payment association, reconciliation, and records.
- Validate data integrity, state transitions, and edge-case handling.
- Log workflow breaks and inconsistencies.

[PHASE 4: PAYMENT & PAYSTACK AUDIT]
- Audit Paystack integration end to end.
- Verify users receive payments directly from clients.
- Confirm payments are not routed through platform owner accounts.
- Validate webhook security, callbacks, error handling, and transaction logging.
- Log compliance, security, and flow correctness.

[PHASE 5: CONFIGURATION & ENVIRONMENT REVIEW]
- Audit environment variables and configuration files.
- Identify insecure, missing, or misconfigured values.
- Validate environment parity across development, staging, and production.

[PHASE 6: DEPLOYMENT & INFRASTRUCTURE AUDIT]
- Review build processes, CI/CD pipelines, and deployment mechanisms.
- Identify stability risks, rollback gaps, and production exposure issues.
- Validate deployment consistency and recovery readiness.

[PHASE 7: UI/UX & DESIGN AUDIT]
- Review all pages for layout, responsiveness, accessibility, and color palette consistency.
- Identify broken UI states and invoice-flow usability issues.

[PHASE 8: DOCUMENTATION AUDIT]
- Review all platform documentation.
- Identify missing, outdated, or incorrect documentation.
- Update or log required documentation changes.

[PHASE 9: REMEDIATION & OPTIMIZATION]
- Fix all identified issues where feasible.
- Complete missing features and implementations.
- Remove unused code and files.
- Refactor for security, performance, scalability, and maintainability.
- Enhance invoice and payment workflows.

[PHASE 10: VALIDATION & REGRESSION TESTING]
- Re-test full invoice lifecycle from creation to payment completion.
- Validate Paystack success, failure, and retry scenarios.
- Confirm no unresolved Critical or High issues remain.

[PHASE 11: FINAL AUDIT CERTIFICATION]
- Generate enterprise audit report including:
  - Executive summary
  - Categorized findings and remediation status
  - Verified Paystack direct-payment compliance
  - Remaining risks (if any)
  - Recommendations for future improvements

FINAL GOAL:
Certify the invoice flow platform as secure, stable, optimized, compliant, and production-ready, with clean code, reliable deployments, and fully functional direct-to-user payment workflows.
